# REPAIR SHOP

Unity 기반 모듈형 미니게임 통합 시뮬레이션

> **Unity Engine을 활용하여 3D 환경과 다수의 2D 미니게임을 통합하고, 디자인 패턴을 적용하여 모듈화된 구조를 구현한 개인 프로젝트입니다.**

## 핵심 개발 내용 (Core Development)

### 1. 디자인 패턴을 활용한 시스템 모듈화

* **템플릿 메서드 패턴:** `Minigame` **추상 클래스**를 설계하여 5종 미니게임의 공통 로직을 분리하고 다형성을 활용하여 확장성 및 유지보수성을 확보했습니다.
* **싱글톤 패턴:** `MinigameManager`, `Score`에 적용하여 전역 데이터 관리를 구현했습니다.
* **오브젝트 풀링 패턴:** 손님과 투사체 객체에 적용하여 생성/삭제 비용을 줄이고 객체를 재활용했습니다.

### 2. 3D 시뮬레이션 환경과 2D 미니게임 통합

* **상호작용 시스템:** Raycast 기반의 3D 상호작용 시스템을 구현하여 아이템 획득 및 배치 기능을 구현했습니다.
* **데이터 기반 동적 호출:** 아이템 정보를 읽어 해당 미니게임을 동적으로 호출하고, `struct`로 난이도 파라미터를 주입하여 난이도별 차등 점수 보상 로직을 구현했습니다.
* **게임 로직 제어:** Enum 기반의 **상태 머신**을 활용하여 손님의 행동을 제어하고, Coroutine 비동기 처리를 활용하여 메인 스레드와 독립적인 점수 연출을 구현했습니다.

## 문제 해결 (Problem Solving)

### 3D-2D 환경 전환 시 입력 충돌 문제 해결 과정

| 문제점 (The Problem) | 해결 전략 (Solution) | 결과 (Result) |
| :--- | :--- | :--- |
| 복합 환경 전환 시 3D 플레이어의 입력 제어권이 해제되지 않아, 2D 미니게임 입력과 3D 이동 조작이 동시 충돌하는 문제 발생 | **상태 기반 제어권 위임 로직 구축:** `Minigame` 클래스의 `GameStart()` 단계에서 3D 제어 객체의 `Pause()`를 호출하여 3D 입력을 강제로 차단 | 3D-2D 환경 간 명확한 **입력 경계**를 확립하여 충돌 오류를 방지했습니다. 안정적인 모드 전환 시스템을 구축하여 **시스템 통합 안정성**을 확보했습니다. |
| | 미니게임 종료 후 `Resume()` 호출로 제어권을 복구하는 로직 적용 | |
